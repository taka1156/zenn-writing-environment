## この記事で伝えたいこと

Node.js に依存せず、Go 製の代替ツールを使って
**commitizen / commitlint / husky** 相当の環境を軽量に構築する方法を紹介します。

## 対象読者

- commitlintやcommitizenをよく使ってる人
- Node.jsでよくcommitlintやcommitizenを使っていて近いものを他言語の開発でも利用したい人
- commitlintを使うためだけにNode.jsを入れたくない人

## 背景・課題

commitlint、commitizenに関して便利ですが、**Node.js が必須**という点が個人的には悩みどころでした。

- Go / Rust / C++ のプロジェクトでも使いたい
- でもNode.jsを入れたくない
- Docker イメージを軽く保ちたい

こうした課題を解決するために**言語非依存**のツールを利用します。

## 環境・前提条件

Docker前提で記載しています。
ホストOSに直接導入する場合でも、curlが叩ける環境であれば、導入可能です。

## 対応方針

まず、Node.js 製ツールと、非言語依存の代替ツールの対応関係について整理します。

|Node.js|言語非依存|
|:---:|:---:|
|husky|[evilmartians/lefthook](https://github.com/evilmartians/lefthook)|
|commitlint|[conventionalcommit/commitlint](https://github.com/conventionalcommit/commitlint)|
|commitizen|[shipengqi/commitizen](https://github.com/shipengqi/commitizen)|

### lefthook

huskyのように`git hook`に引っ掛けてlintを起動させたり、特定のコマンドを走らせるための管理ツール。
lefthook.yamlで設定できます。

### conventionalcommit/commitlint

commitlintの代替ツールです。
.commitlint.yamlで細かい設定できます。

### shipengqi/commitizen

commitizenの代替ツールです。
.czrcというファイルで細かい設定できます。

Conventional　Commit形式を要求されるので人によっては合わないかも？
単にprefixのチェックだけできるかどうかは確認中

### 実装・手順

以下のような記載をDockerfileに追加してください。
※ もし、直接ホスト環境に入れる場合は、commitlintとcommitizenはアーキテクチャを正しいものに差し替えてください。

```Dockerfile
RUN curl -1sLf 'https://dl.cloudsmith.io/public/evilmartians/lefthook/setup.deb.sh' | bash \
     apt update \
     apt install lefthook


RUN curl -fsSL \
  https://github.com/shipengqi/commitizen/releases/download/v2.3.5/commitizen-v2.3.5-linux-amd64.tar.gz \
  | tar -xz -C /usr/local/bin commitizen \
  && chmod +x /usr/local/bin/commitizen \
  && commitizen init

RUN curl -fsSL \
  https://github.com/conventionalcommit/commitlint/releases/download/v0.12.0/commitlint_v0.12.0_linux_amd64.tar.gz \
  | tar -xz -C /usr/local/bin commitlint \
  && chmod +x /usr/local/bin/commitlint

```

## 留意点・デメリット・今後の課題

- shipengqi/commitizenは、Conventional　Commit形式が前提
- prefixだけを対話形式で選ぶことができるかどうかは、確認中
- commitlint のルールも Node.js 版と完全互換ではないため、必要に応じて調整が必要

## まとめ

Go / Rust / C++ などの 非 Node.js 環境でも、軽量にコミットメッセージの品質管理が可能になります。
Docker イメージを肥大化させたくない人にもおすすめです。
