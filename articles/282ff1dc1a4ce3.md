## 概要

Codespaces を使うたびに devcontainer.json を書くのが面倒だったので、自分用の「必要最低限の devcontainer を自動生成する CLI」 を作りました。

### やること

- 以下をスキャフォールディングします。
  - devcontainer.json
  - Dockerfile
  - docker-compose.yaml
- [codespacegen.schema.json](https://github.com/taka1156/codespacegen/blob/master/codespacegen.schema.json)のJSONスキーマに従ってイメージを追加できること

また、以下のような 共通で必要な最低限のパッケージは最初から入れてあります。

- vim
- curl
- tzdata等

### やらないこと（割り切り）

以下は あえて対応していません。

- DB（MySQL / PostgreSQL など）
- ミドルウェア（nginx / Apache）
- mailhog などの開発補助ツール

理由はシンプルで以下のような “スキャフォールディングの本質” に反するためです。
必要なものは 手動で上書きする前提にしています。

- パラメータが多い
- プロジェクトごとに要件が違う
- 使わないことも多い

## Goを選んだ理由

普段はJS/TSメインですが、以下の理由でGoを選びました。

- ランタイムのインストールなしで動く
- バイナリの配布、取り込みが楽
- 簡素な記述ができる

## インストール（curl）

以下のスクリプトを叩くことで、最新リリースを自動でダウンロードし、`/usr/local/bin` に配置できます。

```bash
curl -fsSL https://raw.githubusercontent.com/taka1156/codespacegen/master/scripts/install.sh | bash
```

インストール先を変更する場合:

```bash
curl -fsSL https://raw.githubusercontent.com/taka1156/codespacegen/master/scripts/install.sh | INSTALL_DIR=$HOME/.local/bin bash
```

## 使い方

1. `csg init`を行ってください。
    codespacegen.jsonがカレントディレクトリに生成されるので、それを元に自由に書き換えてください。

    デフォルトは、`codespacegen.json`で認識しますが、`-image-config [ファイル名.json]`とすると任意のファイル名でも動きます。

    :::message
    記載はありませんが、RubyやPHPなど、項目を増やせばいくらでも追加できます。

    それ以外にも、moonbitの例にあるような、インストールが必要なものやgccなどのコンテナに依存しない書き方もできます。
    :::

2.`chmod +x ./codespacegen`などで権限を付与して実行します。
ワークスペース内に.devcontainerディレクトリができるので細かいところは手動で直します。

### cliのオプション

以下、CLIとしてのオプションもつけてみました。

:::details 主なオプション

|     オプション      |             デフォルト              | 説明                                                                                                                             |
| :-----------------: | :---------------------------------: | :------------------------------------------------------------------------------------------------------------------------------- |
|      `-output`      |           `.devcontainer`           | 出力先ディレクトリ                                                                                                               |
|       `-name`       |          *(対話入力必須)*           | プロジェクト名。毎回確認され、`devcontainer.json` の `name` に反映                                                               |
|     `-language`     |      *(対話入力、Enter で空)*       | プログラミング言語。毎回確認される。空の場合は言語固有設定を使わず `alpine:latest` を採用                                        |
|     `-service`      |    *(対話入力、Enter で `app`)*     | docker compose のサービス名。毎回確認され、`devcontainer.json` と `docker-compose.yaml` 両方に反映                               |
| `-workspace-folder` | *(対話入力、Enter で `/workspace`)* | コンテナ内ワークスペースパス。毎回確認される                                                                                     |
|     `-timezone`     |            `Asia/Tokyo`             | コンテナ内のタイムゾーン。Dockerfile の `ENV TZ` と timezone 設定に反映                                                          |
|    `-base-image`    |         *(言語デフォルト)*          | Dockerベースイメージを直接指定。`-language` のデフォルトより優先                                                                 |
|   `-image-config`   |         `codespacegen.json`         | ベースイメージ定義のローカルパスまたは `https://` URL。`install` のみ指定してイメージを省略した場合は `alpine:latest` を自動採用 |
|       `-port`       |  *(対話入力、Enter で ports なし)*  | ポート指定。たとえば `3000` は `3000:3000` に自動正規化され、`8080:3000` も利用できます。毎回確認されます                        |
|   `-compose-file`   |        `docker-compose.yaml`        | Composeファイル名                                                                                                                |
|      `-force`       |               `false`               | 既存ファイルを上書き                                                                                                             |

言語ごとのデフォルトベースイメージ:
※ codespacegen.jsonで上書き可能

- Go: golang:1.24-alpine
- Python: python:3.12-alpine
- Node: node:22-alpine
- Rust: rust:1-alpine

:::
