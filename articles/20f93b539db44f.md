## 動機

devcontainerをスキャフォールドするツールを作ったので、執筆環境もDev containerに閉じ込めて、気軽に執筆できるようにしようと思い行ってみました。

また、一度作れば、GtiHub Codespacesと併用し、ブラウザで執筆することも可能になるのでメリットがあると考えています。

## 環境

- M2 MBA
- devcontainer
  - Node.js v24-alpine

## VS Codeの設定

Dev containerを使ったことがない人は、以下の拡張を入れます。

https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-containers

また、Docker クライアントがない場合そちらのインストールも行います。
※ Dev containerがコンテナ内に開発環境(執筆環境)ごと含めるため。

個人的にDockerクライアントに関しては、以下のどちらかを使えば、問題ないと思います。
企業アカウントによる執筆をする場合は、Rancher DeskTopの方が良いかもしれません。

:::details Rancher Desktop

https://rancherdesktop.io

  ```shell
  brew install --cask rancher
  ```

:::

:::details Docker Desktop

https://www.docker.com

  ```shell
  brew install --cask docker
  ```

:::

## devcontainer環境の作成

※ スキャフォールディングツールを使用しているので自分でゴリゴリ設定したい方は、他の設定方法でご対応をお願いします。

1. 以下のコマンドを実行して、codespacegenを入れます。

    ```shell
    curl -fsSL https://raw.githubusercontent.com/taka1156/codespacegen/master/scripts/install.sh | bash
    ```

1. `csg init` を実行します。
  このコマンドを実行するとカレントディレクトリに**codespacegen.json**というのが生成されます。
  こちらに、今回は**zennの設定**を加えます。

1. **codespacegen.json**に、以下のような設定を追加してください。
   ※ デフォルトだと、rustやnodeの設定がありますが、今回は使わないので削除して問題ないです。

    :::details codespacegen.jsonの設定

      ```json
        {
            "$schema": "https://raw.githubusercontent.com/taka1156/codespacegen/master/codespacegen.schema.json",
            "common": {
                "timezone": "Asia/Tokyo",
                "locale": {
                    "lang": "ja_JP.UTF-8",
                    "language": "ja_JP:ja",
                    "lcAll": "ja_JP.UTF-8"
                },
                "vscodeExtensions": [
                    "MS-CEINTL.vscode-language-pack-ja",
                    "streetsidesoftware.code-spell-checker",
                    "username.errorlens"
                ]
            },
            "langs": [
              {
                "profileName": "node:zenn"
                "image": "node:24-alpine",
                "runCommand": "yarn global add zenn-cli",
                "vscodeExtensions": [
                  "davidanson.vscode-markdownlint",
                  "yzhang.markdown-all-in-one",
                  "zenn.zenn-preview"
                ]
              }
            ]
        }
      ```

    :::

1. `csg`を実行して対話質問に答えて.devcontainerファイルを生成します。

    :::details 対話形式の内容

    ```shell
    root@c648065dcef1 /workspace/ (draft*%)+# csg
    プロジェクト名を入力してください（必須）: sample
    言語を入力してください（未入力で alpine 固定）: node:zenn
    ワークスペースを入力してください（未入力で /workspace）: 
    サービス名を入力してください（未入力で app）: 
    公開ポートを入力してください (例: 3000 または 3000:3000、不要ならEnter): 
    タイムゾーンを入力してください（未入力で Asia/Tokyo）: 
    Codespaceファイルを /workspace/.devcontainer に生成しました
    ```

    :::

1. `command`+`shift`+`P`(Mac) / `Ctrl`+`Shift`+`P`(Windows) を押してコマンドパレットを開き、`Reopen in Container` と入力します。

   その後、コンテナのビルドが始まり、別ウィンドウで開き直されます。

1. ターミナルが使えるようになったら、校正関連のlintをインストールします。

    :::details textlint、markdownlintの追加

    ```shell
    npm install -D @textlint-plugin-markdown \
          markdownlint-cli \
          markdownlint \
          textlint \
          textlint-filter-rule-comments \
          textlint-rule-preset-ja-technical-writing \
          textlint-rule-spellcheck-tech-word
        
      or
        
    yarn add -D @textlint-plugin-markdown \
          markdownlint-cli \
          markdownlint \
          textlint \
          textlint-filter-rule-comments \
          textlint-rule-preset-ja-technical-writing \
          textlint-rule-spellcheck-tech-word
    ```

    :::

1. `.devcontainer/devcontainer.json`の`postCreateCommand`にモジュールインストールコマンドを追加します。
   これを行うことでコンテナの初回作成時にNode モジュールのインストールを自動で行うため、スムーズな執筆が可能になります。

    :::details .devcontainer.json

    ```jsonc
    {
      "$schema": "https://raw.githubusercontent.com/microsoft/vscode/main/extensions/configuration-editing/schemas/devContainer.vscode.schema.json",
      "name": "zenn-writing-environment",
      "service": "app",
      "workspaceFolder": "/workspace",
      "dockerComposeFile": "docker-compose.yaml",
      "postCreateCommand": "yarn install" /* npm installなど適宜読み換え */
      "customizations": {
        "vscode": {
          "settings": {
            "terminal.integrated.defaultProfile.linux": "bash",
          },
          "extensions": [
            "MS-CEINTL.vscode-language-pack-ja",
            "streetsidesoftware.code-spell-checker",
            "username.errorlens",
            "davidanson.vscode-markdownlint",
            "yzhang.markdown-all-in-one",
            "zenn.zenn-preview"
          ]
        }
      }
    }
    ```

    :::

1. 設定が一通り終わったら設定をコミットして、pushします。

## markdownlintの設定

  :::details .markdownlint.json

- markdownlint-cli
- markdownlint

  ```json
  {
    "$schema": "https://raw.githubusercontent.com/DavidAnson/markdownlint/refs/heads/main/schema/markdownlint-config-schema.json",
    "default": true,
    "MD013": false,
    "MD024": {
        "siblings_only": true
    },
    "MD034": false,
    "MD041": false
  }
  ```

  :::

- 全てのルールを全許可  
- 1行の文字数制限を無効
  textlintで縛る
- 先頭がh1を無効
  見出し..いわゆる「Hero Area」があるため
- 同じ見出し名の重複禁止
  リストの子見出しは許可
- リンクを<>で囲うルールを無効化
  (リンクを改行スペースで挟むと、OGP付き表示をする機能がZennにあるため)

## textlintの設定

  :::details .textlintrc.json

- textlint-filter-rule-comments
- textlint-rule-preset-ja-technical-writing
- textlint-rule-spellcheck-tech-word

  ```jsonc
  {
    "plugins": {
      "@textlint/markdown": true
    },
    "filters": {
      "comments": true
    },
    "rules": {
      "preset-ja-technical-writing": {
        "ja-no-mixed-period": false, /* detailsの拡張記載が使えないので無効化 */
        "no-mix-dearu-desumasu": {
          "prefer": "desumasu"
        },
        "ja-no-weak-phrase": false,
        "ja-no-redundant-expression": true,
        "no-exclamation-question-mark": false,
        "no-doubled-joshi": {
          "min_interval": 2
        },
        "sentence-length": {
          "max": 100
        }
      },
      "spellcheck-tech-word": true
    }
  }
  ```

  :::

- 「ら抜き言葉」や「助詞を2回以上重ねる」ことは禁止。  
- 冗長な言い回しはしない。(「~という風に」=> 「〜のように」)  
- いいきりの形であること。  
- 「！」や「?」は基本的には禁止にしない。  

## 記事の作成

1. 以下のコマンドでzennの執筆環境を作成します。  
  `yarn zenn init`
1. zenn new:articleコマンドで記事の雛形を作成します。  
  `yarn zenn new:article`
1. markdown で記事を書いていき、適宜リンターに従って校正します.
1. 書き終わったら、以下のコマンドでpreview
   `yarn zenn preview`

## 校正まで終わった後

1. frontmatterを`pablish:true`に変更します。
1. 更新内容をコミットしておきます。
1. GitHub リポジトリをZennと連携します。
1. master/mainにpush => Zennが自動デプロイします。

## その他: snippets追加

`.vscode/markdown.code-snippets`に以下のような記載を追加すると、アコーディオンなどの拡張構文がすぐに出せるようになります。

```md
:::details 折りたたみテキスト
  アコーディオン（トグル）
:::
```

1. `.vscode/markdown.code-snippets`の作成

    :::details markdown.code-snippets

    ```json
    {
        "zenn warning": {
            "prefix": "warning",
            "body": [
                ":::message $1",
                "$2",
                ":::"
            ],
            "description": "Zenn snippet for warning"
        },
        "zenn alert": {
            "prefix": "alert",
            "body": [
                ":::message alert $1",
                "$2",
                ":::"
            ],
            "description": "Zenn snippet for alert"
        },
        "zenn details": {
            "prefix": "details",
            "body": [
                ":::details $1",
                "$2",
                ":::"
            ],
            "description": "Zenn snippet for details "
        },
    }
    ```

    :::

1. `.devcontainer/devcontainer.json`にマークダウンのサジェスト設定を追加します。
   ※ デフォルト設定だと、マークダウンではサジェストが出ないためです。

    :::details .devcontainer.json

    ```jsonc
    {
      "$schema": "https://raw.githubusercontent.com/microsoft/vscode/main/extensions/configuration-editing/schemas/devContainer.vscode.schema.json",
      "name": "zenn-writing-environment",
      "service": "app",
      "workspaceFolder": "/workspace",
      "dockerComposeFile": "docker-compose.yaml",
      "postCreateCommand": "yarn install",
      "customizations": {
        "vscode": {
          "settings": {
            "terminal.integrated.defaultProfile.linux": "bash",
            "[markdown]": {
              "editor.quickSuggestions": { /* この記載を追加 */
                "other": "on",
                "comments": "off",
                "strings": "off"
              }
            }
          },
          "extensions": [
            "MS-CEINTL.vscode-language-pack-ja",
            "streetsidesoftware.code-spell-checker",
            "username.errorlens",
            "davidanson.vscode-markdownlint",
            "yzhang.markdown-all-in-one",
            "zenn.zenn-preview"
          ]
        }
      }
    }
    ```

    :::

## あとがき

Dev containerを用意しておくことで、ちょっとした校正や修正程度なら、Github Codespacesを利用し、ウェブ上で行うことができます。

ローカル環境でも開発環境ごと、Gitに閉じ込めることができるので、Docker クライアントとVS Codeさえあれば、すぐ続きを始められるようになりました。
