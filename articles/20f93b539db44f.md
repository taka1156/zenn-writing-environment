## 動機

devcontainerをスキャフォールドするツールを作ったので、執筆環境もDev containerに閉じ込めて、気軽に執筆できるようにしようと思い行ってみました。

また、一度作れば、GtiHub CodespacesでWeb上で開発することも可能になるのでそういう意味でもメリットがあると考えてやってみました。

## 環境

- M2 MBA
- devcontainer
  - Node.js v24-alpine

## VS Codeの設定

Dev containerを使ったことがない人は、以下の拡張を入れます。

https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-containers

また、Docker クライアントがない場合そちらのインストールも行います。
※ Dev containerがコンテナ内に開発環境(執筆環境)ごと含めるため。

個人的に以下のどちらかを使えば、問題ないと思いますが、

https://rancherdesktop.io

  ```shell
  brew install --cask rancher
  ```

:::

::: details Docker Desktop

https://www.docker.com

  ```shell
  brew install --cask docker
  ```

:::

## devcontainer環境の作成

※ スキャフォールドしてるので自分でゴリゴリ設定した方は、別で設定ををお願いします。

1. 以下のコマンドを実行して、codespacegenを入れる。
  `curl -fsSL https://raw.githubusercontent.com/taka1156/codespacegen/master/scripts/install.sh | bash`

1. `csg init` を実行する。
   codespacegen.jsonというのが吐き出されます。
   こちらに、今回はzennの設定を加えます。

1. codespacegen.jsonにような設定を追加してください。
   ※ デフォルトだと、rustやnodeの設定がありますが、今回は使わないので削除して問題ないです。

    ::: details codespacegen.jsonの設定

      ```json
        {
            "$schema": "https://raw.githubusercontent.com/taka1156/codespacegen/refs/heads/master/codespacegen.schema.json",
            "common": {
                "timezone": "Asia/Tokyo",
                "locale": {
                    "lang": "ja_JP.UTF-8",
                    "language": "ja_JP:ja",
                    "lcAll": "ja_JP.UTF-8"
                },
                "vscodeExtensions": [
                    "MS-CEINTL.vscode-language-pack-ja",
                    "streetsidesoftware.code-spell-checker",
                    "username.errorlens"
                ]
            },
              "node:zenn": {
              "image": "node:24-alpine",
              "install": "yarn global add zenn-cli",
              "vscodeExtensions": [
                "davidanson.vscode-markdownlint",
                "yzhang.markdown-all-in-one",
                "zenn.zenn-preview"
              ]
            }
        }
      ```

    :::

1. `csg`を実行して対話質問に答えて設定

    ::: details codespacegen.jsonの設定

    ```shell
    root@c648065dcef1 /workspace/sandbox (draft*%) 
    +# csg

    プロジェクト名を入力してください（必須）: sample
    言語を入力してください（未入力で alpine 固定）: node:zenn
    ワークスペースを入力してください（未入力で /workspace）: 
    サービス名を入力してください（未入力で app）: 
    公開ポートを入力してください (例: 3000 または 3000:3000、不要ならEnter): 
    タイムゾーンを入力してください（未入力で Asia/Tokyo）: 
    Codespaceファイルを /workspace/.devcontainer に生成しました
    ```

    :::
1. 生成が終わったら細かいlint設定など追加したい項目を直接Dockerfileに記載して快適にさせていきます。

1. 校正関連の設定をインストールします。

    ::: details textlint、markdownlintの追加

    ```shell
    yarn add zenn-cli \
          @textlint-plugin-markdown \
          markdownlint-cli \
          textlint \
          textlint-rule-ja-no-mixed-period \
          textlint-rule-no-doubled-joshi \
          textlint-rule-no-dropping-the-ra \
          textlint-rule-preset-ja-technical-writing \
          textlint-rule-spellcheck-tech-word
    ```

    :::

1. 設定が一通り終わったら環境を設定をコミットして、pushします。

## markdownlintの設定

  ::: details .markdownlint.json

- markdownlint-cli
- markdownlint

  ```json
  {
    "$schema": "https://raw.githubusercontent.com/DavidAnson/markdownlint/refs/heads/main/schema/markdownlint-config-schema.json",
    "default": true,
    "MD013": false,
    "MD024": {
        "siblings_only": true
    },
    "MD033": {
        "allowed_elements": [
            "details",
            "summary"
        ]
    },
    "MD034": false,
    "MD041": false
  }
  ```

  :::

- 全てのルールを全許可  
- 1行の文字数制限無効
  textlintで縛る
- インラインHTMLはdetails - summary以外禁止
- 先頭がh1を無効
  見出し..いわゆる「Hero Area」があるため
- 同じ見出し名の重複禁止
- リンクを<>で囲うルールを無効化
  (リンクを改行スペースで挟むとOGPありで表示する機能がZennにあるため)

## textlintの設定

  ::: details .textlintrc.json

- textlint-rule-ja-no-mixed-period
- textlint-rule-no-doubled-joshi
- textlint-rule-no-dropping-the-ra
- textlint-rule-preset-ja-technical-writing
- textlint-rule-spellcheck-tech-word

  ```json
  {
      "plugins": {
          "@textlint/markdown": {
              "extensions": [".md"]
          }
      },
      "rules": {
          "preset-ja-technical-writing": {
              "ja-no-weak-phrase": false,
              "ja-no-redundant-expression": true,
              "no-exclamation-question-mark": false,
              "sentence-length": {
                "max": 100
              }
          },
          "ja-no-mixed-period": {
              "periodMark": "。"
          },
          "no-dropping-the-ra": true,
          "no-doubled-joshi": {
              "min_interval": 1
          },
          "spellcheck-tech-word": true,
          "filters": {
              "comments": true
          }
      }
  }
  ```

  :::

- 「ら抜き言葉」や「助詞の2回以上重ねる」ことは禁止。  
- 冗長な言い回しはしない。(「~という風に」=> 「〜のように」)  
- 末尾を「。」で終らせる。  
- いいきりの形であること。  
- 「！」や「?」は基本的には禁止にしない。  

## 記事の作成

1. 以下のコマンドでzennの執筆環境を作成する。  
  `yarn exec zenn init`
1. zenn new:articleコマンドで記事の雛形を作成する。  
  `yarn exec zenn new:article`
1. markdown で記事を書いていき、適宜リンターに従って校正を行う.
1. 書き終わったら、以下のコマンドでpreview
   `yarn exec zenn preview`

## 校正まで終わった後

1. GitHub RepositoryをZennと連携する。  
1. frontmatterを`pablish:true`にかえる。
1. Repositoryにpush => Zennが自動デプロイ。

## あとがき

Dev containerを用意しておくと、ちょっとした校正や修正程度なら、Github Codespacesを利用して、WebのVS Code経由で、iPadなどでも編集ができるようになるのでdevcontainerぜひ使ってください。

ローカル環境でも開発環境ごと、gitに閉じ込めることができるので、Docker クライアントとVS Codeさえあれば、すぐ続きを始められます。
