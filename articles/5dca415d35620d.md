## この記事で伝えたいこと

Ansibleを利用して以下を構築し、PortainerのStack機能を利用して、Github Actionsからのデプロイを可能にします。

- Docker CE
- Docker Compose
- Portainer CE
- Cloudflare Tunnel

## 対象読者

- 個人サービスを安価に運用したい人
- インフラ構築手順書を書くのがだるくてIaCでやりたい人

## 背景

vps環境下でも、DockerやGitHub Actionsを利用したリリース環境を作りたいと思ったため。

## やったこと

Dev Container内で安全にansibleやSSHを実行できる環境を作りました。
具体的には、以下の通りです。

- 対話形式による接続情報の設定
  - Cloudflare Tunnel の設定
  - Copilot向けのAI設定をオプトイン
- 設定したenvファイルを利用して、ansibleのtarget設定(hosts.ini)を自動生成
- secretlintを利用して環境変数が意図せず入ることを防ぐ

## 環境・前提条件

- Docker 踏み台 (Dev containers)
  - Ubuntu

- 必須モジュール
  - cloudflared
  - ansible
  - ansible-lint
  - secretlint
  
- 本番環境
  - ConoHa vps

https://github.com/taka1156/portainer-stack

## 手順
<!-- コードブロックの前に「何をするか」、後に「なぜそうしたか」を書く。 -->

```shell
# .envを対話形式で作成
make setup

# .envの情報を元に hosts.ini を作成してdry-run
make setup-ansible ARGS="--check --diff"
```

## 結果

- [PortainerのStack機能で、Git RepositoryとWebhooks](https://docs.portainer.io/user/docker/stacks/add#option-3-git-repository)を利用することで、GitHub Actions経由でwebhookを叩いてデプロイが可能
- Portainerのダッシュボードを利用して、SSHで接続せずにコンテナの管理やデプロイ設定が可能です。
- 必要に応じて、PortainerのダッシュボードをCloudflare Accessでガードが可能です。(Cloudflareのダッシュボードから設定)

## 留意点

Cloudflare Tunnelを使う前提の構成なので使わない場合は、nginxやTraefikなどのリバースプロキシサーバーが必要です。
その場合は、Portainerではなく、Dokployを利用すると運用しやすくなります。
※ Traefikが組み込まれているため、Let's Encryptが有効化されています。また、DokployのGUIでリバースプロキシ設定ができるなどの恩恵が得られます。

https://dokploy.com

## まとめ

Portainerと、Cloudflareを利用して安全にコンテナを利用したサービスをデプロイする環境ができました。
今後やっていくこととしては、以下を考えており、より品質を高めていければと考えています。

- Ansible Molecule を利用し、テストを実施
- Dockerを利用したサンドボックス環境を作り、実際にAnsibleのコードが走る実験環境をローカルで構築
